vue国际化快速实现方案

更新于2021-06-02|vue
|阅读量:

vue 做国际化，一般是搭配 i18n，但是多种语言的适配，一个个的手敲实在让人感到崩溃，那么有没有方便快捷而不失准确度的方案呢？

找了好久，特此记录

  • 百度了一大波，发现了三种方案 :
    1. 谷歌网页翻译
    2. 正则匹配，vue 节点更新注入
    3. 本地 i18n 适配

第一种，谷歌网页翻译

借用谷歌翻译 api，进行整个网页的翻译，看似完美，实则很坑。

  • 内陆连接谷歌翻译略慢，就算按照大佬的方案 copy 部分源码，进行本地加载提速，依然需要好几秒
  • 谷歌翻译无法翻译内嵌元素，包括弹出层、提示框，以及 echarts 等定制元素

综上所述，不实用

第二种，正则匹配，搭配 vue 更新注入结果

这种方式，更适合做简单的简繁切换，在合适的时机注入。

  1. 首先，我们建立一个 zh.txt 和 en.txt，保存备用
  2. 建立备份或者上传 commit
  3. 打开谷歌翻译或者其他翻译
  4. 自定义大写快捷键，比如我的是 ctrl+shift+u

正式开始

  1. 分离 vue 的模板和 js，备用，正则匹配
  2. 全选正则匹配内容，备份中文；翻译之，备份英文
  3. 使用快捷键批量转换大小写，命名变为驼峰
  4. 批量替换，并拼接成 json 或其他格式
  5. 使用 i18n ally 插件进行批量翻译
  6. 重复以上动作，js 中的则加上 this 即可

下面说说坑：

  1. 使用 i18n ally 时，翻译的 json 文件的 key 值最好不要采取_或者-连接符或者其他连接符，插件有可能读取错误，会删掉你的文件中的字段，导致重写…
  2. 正则匹配不要太追求完美，能用就行
